Bandit 靶场练习记录

Feb. 17, 2020 |

Wargames是OverTheWire社区提供的一个帮助学习安全知识和实践的一个平台，提供了各式各样的项目模块，可以在其中练级基本的安全技能。 Bandit是Wargames最基础的一个模块，用来练习最基本的Linux操作知识，同时适应Wargames的模式。

Wargames bandit

Level 0~7

Level 0:

Level就是测试一下ssh客户端，直接连接到bandit.labs.overthewire.org:2220就行了。

Level 0->1:

The password for the next level is stored in a file called readme located in the home directory. Use this password to log into bandit1 using SSH. Whenever you find a password for a level, use SSH (on port 2220) to log into that level and continue the game. 密码在home的readme中，那么很简单，进入Home目录，输出文件即可。

$ cd ~
$ cat readme

Level 1->2：

The password for the next level is stored in a file called - located in the home directory。 密码在 “-“里，直接使用cat发现它停了下来，猜测一下，因为linux命令的短选项通常是以一个“-”开始，因此其可能将其作为某个选项的开头，所以无法作为文件打开。那么就要想方法绕过这个限制。

bandit1@bandit:~$ cat ./-
CV1DtqXWVFXTvM2F0k09SHz0YwRINYA9

可以了。 *nix常用特殊字符

Level 2->3：

The password for the next level is stored in a file called spaces in this filename located in the home directory。 密码在“spaces in this file”文件里，这个文件的特别之处就在与有空着，直接输入会被命令视为多个文件，因此使用双引号将文件名引起来。

bandit2@bandit:~$ cat "spaces in this filename" 
UmHadQclWmgdLOKQ3YNgjWxGoRMb5luK

Level 3->4：

The password for the next level is stored in a hidden file in the inhere directory. 密码在一个隐藏文件里。Linux下以“.”开头的文件会被隐藏，使用 ls -a 选项可以查看隐藏文件。所以，先进入inhere目录，使用ls -a查看文件。

bandit3@bandit:~$ cd inhere/
bandit3@bandit:~/inhere$ ls -a
.  ..  .hidden
bandit3@bandit:~/inhere$ cat .hidden 
pIwrPrtPN36QITSp3EQaw936yaFoFgAB

Level 4->5：

The password for the next level is stored in the only human-readable file in the inhere directory. Tip: if your terminal is messed up, try the “reset” command. 密码在一个人类可读的文件里，打开inhere文件，发现里面有10个文件，题目说是人类可读，那就应该是个文本文件，使用file查看文件类型。当然这里的文件都以“-”开头，所以还需要前面的方法来绕过命令执行：

Level 5->6：

The password for the next level is stored in a file somewhere under the inhere directory and has all of the following properties:

  • human-readable
  • 1033 bytes in size
  • not executable

密码在inhere目录的某个子目录里，而且具有那几样属性。很显然，是要使用find命令并通过属性来筛选。使用man find可以查看find的各个选项和参数。

bandit5@bandit:~/inhere$ find -size 1033c -type f ! -executable 
./maybehere07/.file2
bandit5@bandit:~/inhere$ cat ./maybehere07/.file2
DXjZPULLxYr17uwoI01bNLQbtFemEgo7

Level 6->7:

The password for the next level is stored somewhere on the server and has all of the following properties:

  • owned by user bandit7
  • owned by group bandit6
  • 33 bytes in size

和上题类似，仍然是查询相关属性。因为这次范围是服务器:

bandit6@bandit:/$ find / -user bandit7 -group bandit6 -size 33c 2>/dev/null
./var/lib/dpkg/info/bandit7.password
bandit6@bandit:/$ cat ./var/lib/dpkg/info/bandit7.password
HKBPTKQnIay4Fw76bEy8PVxKEDQRKTzs

因为权限原因会提示很多警告，为方便查找，将他们重定向值黑洞，然后就能找到密码了。

Level 8~13

Level 7->8：

The password for the next level is stored in the file data.txt next to the word millionth。 密码在data.txt并且紧邻着millionth。那么很明显了，使用grep命令。

bandit7@bandit:~$ cat data.txt | grep millionth

Level 8->9：

The password for the next level is stored in the file data.txt and is the only line of text that occurs only once。 密码所在行在data.txt中只出现一次。提示中有一个uniq命令，先查询一下：

uniq - report or omit repeated lines 就是他了。uniq会忽略重复的行，但是他们必须是相邻的，我们还需要找到方法将相同的行放在一起。

bandit8@bandit:~$ sort data.txt | uniq -u

The password for the next level is stored in the file data.txt in one of the few human-readable strings, beginning with several ‘=’ characters. strings命令在对象文件或二进制文件中查找可打印的字符串。字符串是4个或更多可打印字符的任意序列，以换行符或空字符结束。

Level 10->11：

The password for the next level is stored in the file data.txt, which contains base64 encoded data。 密码在data.txt中，但是这个文件是使用base64加密的，使用base64解密即可。

bandit10@bandit:~$ base64 -d data.txt The password is IFukwKGsFW8MOq3IRFqrxE1hxTNEbUPR

Level 11->12：

The password for the next level is stored in the file data.txt, where all lowercase (a-z) and uppercase (A-Z) letters have been rotated by 13 positions。 密码在data.txt中，但是所有的字母都偏移了13个位置。使用tr命令可以进行字符替换。

bandit11@bandit:~$ cat data.txt | tr [a-zA-Z] [n-za-mN-ZA-M]The password is 5Te8Y4drgCRfCx8ugdwuEX8KFC6k2EUu

Level 14~21

Level 13->14：

The password for the next level is stored in /etc/bandit_pass/bandit14 and can only be read by user bandit14. For this level, you don’t get the next password, but you get a private SSH key that can be used to log into the next level. Note: localhost is a hostname that refers to the machine you are working on. 这一次不是使用密码登录，而是使用ssh key登录。可以发现sshkey就在在当前目录下，直接使用ssh登陆即可。

bandit13@bandit:~$ ssh localhost -l bandit14 -i ./sshkey.private Could not create directory '/home/bandit13/.ssh'.The authenticity of host 'localhost (127.0.0.1)' can't be established.ECDSA key fingerprint is SHA256:98UL0ZWr85496EtCRkKlo20X3OPnyPSB5tB5RPbhczc.Are you sure you want to continue connecting (yes/no)? yes

Level 14->15：

The password for the next level can be retrieved by submitting the password of the current level to port 30000 on localhost. 使用当前级别的密码登录从本地登录到30000端口。

bandit14@bandit:~$ telnet -l bandit15 localhost 30000Trying 127.0.0.1...Connected to localhost.Escape character is '^]'.4wcYUJFw0k0XLShlDzztnTBHiqxU3b3eCorrect!BfMYroe26WYalil77FoDi9qh59eK5xNrConnection closed by foreign host.

Level 16：

The password for the next level can be retrieved by submitting the password of the current level to port 30001 on localhost using SSL encryption. Helpful note: Getting “HEARTBEATING” and “Read R BLOCK”? Use -ign_eof and read the “CONNECTED COMMANDS” section in the manpage. Next to ‘R’ and ‘Q’, the ‘B’ command also works in this version of that command… 这次要求使用ssl加密连接，根据给出的命令提示，猜测应该是使用openssl，先使用man openssl查看一下用法。 值得注意的是s_client，但是没有给出具体用法，继续使用帮助信息，man s_client。可以看到，使用 -connect可以指定连接的主机和端口，那么payload就应该是：

bandit15@bandit:~$ openssl s_client -connect localhost:30001 ...---BfMYroe26WYalil77FoDi9qh59eK5xNrCorrect!cluFn7wTiGryunymYOu4RcffSxQluehdclosed

可以看到返回了一个密码。 补充：提示信息没有看的太明白，似乎是出现问题的话可以加上 -ign_eof选项，并且还有一些选项可以使用；这里 -ign_eof也是s_client的一个选项，后面的没有看懂。 -ign_eof是忽略文件结尾EOF，但是我加上之后没有什么变化，还需要学习。

Level 17：

The credentials for the next level can be retrieved by submitting the password of the current level to a port on localhost in the range 31000 to 32000. First find out which of these ports have a server listening on them. Then find out which of those speak SSL and which don’t. There is only 1 server that will give the next credentials, the others will simply send back to you whatever you send to it. 简单来说仍然是将当前的密码提交到某个端口。需要扫描31000到32000端口，查看哪些端口是开放的，那些是加密的，只有一个会返回下一级的密码，其他的都仍然返回输入的东西。 很显然，需要先使用Nmap。

bandit16@bandit:~$ nmap -sV -p31000-32000 localhost

Starting Nmap 7.40 ( https://nmap.org ) at 2020-02-18 05:13 CET
Nmap scan report for localhost (127.0.0.1)
Host is up (0.00032s latency).
Not shown: 999 closed ports
PORT      STATE SERVICE     VERSION
31518/tcp open  ssl/echo
31790/tcp open  ssl/unknown

显然，访问31518会返回你输入的内容，我们要访问31790端口。

bandit16@bandit:~$ openssl s_client -connect localhost:31790

根据提示输入密码后返回的是一个密钥，所以应该是使用密钥登陆。先将密钥保存下来，然后使用之前的方法登录：

bandit16@bandit:/tmp/name12341$ ssh -i sshkey.private bandit17@localhost

Level 21：

There is a setuid binary in the homedirectory that does the following: it makes a connection to localhost on the port you specify as a commandline argument. It then reads a line of text from the connection and compares it to the password in the previous level (bandit20). If the password is correct, it will transmit the password for the next level (bandit21). NOTE: Try connecting to your own network daemon to see if it works as you think. 题目要求使用home二进制文件连接到本地的一个端口，然后会对比bandit20的密码，如果相同就会向其发送bandit21的密码。所以，我们需要用nc监听本地的一个端口，然后使用本地的文件访问这个端口，最后接收到密码：

# 先查询一下端口开放情况bandit20@bandit:~$ nmap localhost...Not shown: 997 closed portsPORT      STATE SERVICE22/tcp    open  ssh113/tcp   open  ident30000/tcp open  ndmps...# 使用nc监听端口bandit20@bandit:~$ echo "GbKksEFF4yrVs6il55v6gwY5aVje5f0j" | nc -l -p 40000# 使用本地下的文件连接bandit20@bandit:~$ ./suconnect 40000Read: GbKksEFF4yrVs6il55v6gwY5aVje5f0jPassword matches, sending next password# 接受密码bandit20@bandit:~$ echo "GbKksEFF4yrVs6il55v6gwY5aVje5f0j" | nc -l -p 40000gE269g2h3mw3pwgrj0Ha9Uoqen1c9DGr

Level 22~26

Level 22：

A program is running automatically at regular intervals from cron, the time-based job scheduler. Look in /etc/cron.d/ for the configuration and see what command is being executed. 题目说明这里有一个使用cron定期运行的任务，查看/etc/cron.d/来查看具体的命令：

bandit21@bandit:~$ ls /etc/cron.d/atop  cronjob_bandit22  cronjob_bandit23  cronjob_bandit24bandit21@bandit:~$ cat /etc/cron.d/cronjob_bandit22@reboot bandit22 /usr/bin/cronjob_bandit22.sh &> /dev/null* * * * * bandit22 /usr/bin/cronjob_bandit22.sh &> /dev/null

可以看出这个任务是运行/usr/bin/cronjob_bandit22.sh，将其输出重定向至/dev/null中，我们看看这个脚本具体是什么：

bandit21@bandit:~$ cat /usr/bin/cronjob_bandit22.sh#!/bin/bashchmod 644 /tmp/t7O6lds9S0RqQh9aMcz6ShpAoZKF7fgvcat /etc/bandit_pass/bandit22 > /tmp/t7O6lds9S0RqQh9aMcz6ShpAoZKF7fgv

可以看出这个脚本将bandit22的密码输入到了/tmp/t706这个文件里，并且将这个文件的权限定义为所有人可读，那么直接从中就可以得到bandit22的密码。

bandit21@bandit:~$ cat /tmp/t7O6lds9S0RqQh9aMcz6ShpAoZKF7fgvYk7owGAcWjwMVRwrTesJEwB7WVOiILLI

Level 23：

A program is running automatically at regular intervals from cron, the time-based job scheduler. Look in /etc/cron.d/ for the configuration and see what command is being executed. NOTE: Looking at shell scripts written by other people is a very useful skill. The script for this level is intentionally made easy to read. If you are having problems understanding what it does, try executing it to see the debug information it prints. 和上题给出的差不多，先读取一下这个任务是干什么：

bandit22@bandit:~$ cat /etc/cron.d/cronjob_bandit23@reboot bandit23 /usr/bin/cronjob_bandit23.sh  &> /dev/null* * * * * bandit23 /usr/bin/cronjob_bandit23.sh  &> /dev/nullbandit22@bandit:~$ cat /usr/bin/cronjob_bandit23.sh #!/bin/bashmyname=$(whoami)mytarget=$(echo I am user $myname | md5sum | cut -d ' ' -f 1)echo "Copying passwordfile /etc/bandit_pass/$myname to /tmp/$mytarget"cat /etc/bandit_pass/$myname > /tmp/$mytarget

和上题一样，我们先阅读一下这个脚本：首先将用户赋值给myname，然后将“I am user $myname"进行加密在切割，存储到mytarget中，然后将myname的密码输入到mytarget中。 所以，我们只要使用bandit23重复这个过程，得到bandit23对应的target就可以获得其密码。

bandit22@bandit:~$ echo I am user bandit23 | md5sum | cut -d ' ' -f 18ca319486bfbbc3663ea0fbe81326349bandit22@bandit:~$ cat /tmp/8ca319486bfbbc3663ea0fbe81326349jc1udXuA1tiHqjIsL8yaapX5XIAI6i0n

Level 24：

A program is running automatically at regular intervals from cron, the time-based job scheduler. Look in /etc/cron.d/ for the configuration and see what command is being executed. NOTE: This level requires you to create your own first shell-script. This is a very big step and you should be proud of yourself when you beat this level! NOTE 2: Keep in mind that your shell script is removed once executed, so you may want to keep a copy around… 题目要求还是一样的…
